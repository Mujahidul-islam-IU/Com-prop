# DigitalOcean Deployment Guide: CRE Scraper & Enquiry Automator

This guide details how to deploy the **LBKN CRE Automator** application on a DigitalOcean Ubuntu VPS in the Sydney region.

---

## 1. DigitalOcean Droplet Selection

*   **Region:** Sydney (`syd1`) — *Crucial for low latency and local Australian listing speeds.*
*   **Operating System:** Ubuntu 24.04 LTS (x64)
*   **Size:** **Basic CPU - Shared Options** ➔ **Regular SSD** or **Premium Intel/AMD**
    *   *Minimum Recommended:* **$12/month (2 GB RAM / 2 vCPUs)**.
    *   *Why?* Google Chrome requires significant RAM. A 1 GB Droplet will likely trigger out-of-memory crashes while running scraping processes. The client's droplet (2 vCPU, 2 GB RAM) is a great choice.

---

## 2. Server Installation & Configuration

Connect to your Droplet via SSH:
```bash
ssh root@YOUR_DROPLET_IP
```

### Step 2.1: Update System Packages
```bash
sudo apt update && sudo apt upgrade -y
```

### Step 2.2: Install Python and PIP
```bash
sudo apt install -y python3 python3-pip python3-venv git
```

### Step 2.3: Install Google Chrome (Stable)
Since `undetected-chromedriver` requires a real Chrome installation, install Google Chrome:
```bash
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install -y ./google-chrome-stable_current_amd64.deb
```
Verify the installation:
```bash
google-chrome --version
```

### Step 2.4: Install Xvfb (Virtual Display Framebuffer)
`undetected-chromedriver` behaves best when running with a virtual graphical display to bypass advanced bot detection algorithms.
```bash
sudo apt install -y xvfb
```

## 3. Clone Repository & Setup Project

### Step 3.1: Clone Project Files
Create a deployment folder in `/var/www/` or your home directory:
```bash
cd /var/www
git clone <YOUR_GIT_REPO_URL> cre-automator
cd cre-automator
```

### Step 3.2: Set Up Virtual Environment
Creating a virtual environment ensures Python package isolation:
```bash
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

### Step 3.3: Configuration Settings
Copy the example config and edit it with your credentials:
```bash
cp backend/config.example.py backend/config.py
nano backend/config.py
```
> [!IMPORTANT]
> If you are using Google Sheets integration, make sure you copy your Google Cloud `service_account.json` into the `backend/` directory as well.

## 4. Run & Monitor Server in Production

We will use **PM2** (via Node/NPM) or **Systemd** to keep our FastAPI app running continuously. 

### Option A: Using PM2 (Recommended for Process Management)

1.  **Install Node.js & PM2:**
    ```bash
    sudo apt install -y nodejs npm
    sudo npm install -y -g pm2
    ```
2.  **Create a Start Script:**
    PM2 needs a wrapper to run the FastAPI app under the virtual display environment. Create a script called `start_prod.sh` in the root folder:
    ```bash
    nano start_prod.sh
    ```
    Paste the following content inside it:
    ```bash
    #!/bin/bash
    source /var/www/cre-automator/venv/bin/activate
    # Run uvicorn under Xvfb virtual frame buffer
    xvfb-run --server-args="-screen 0 1920x1080x24" uvicorn backend.api:app --host 0.0.0.0 --port 8000 --workers 1
    ```
    Make the script executable:
    ```bash
    chmod +x start_prod.sh
    ```
3.  **Start with PM2:**
    ```bash
    pm2 start ./start_prod.sh --name cre-automator
    ```
4.  **Configure Startup Script:**
    Ensure the process starts automatically if the server reboots:
    ```bash
    pm2 startup
    # Copy/paste the systemd command generated by the command above
    pm2 save
    ```

## 5. Security & Reverse Proxy (Nginx)

To secure the app and run it on port `80` (HTTP) or `443` (HTTPS) instead of port `8000`, configure Nginx.

### Step 5.1: Install Nginx
```bash
sudo apt install -y nginx
```

### Step 5.2: Configure Virtual Host
```bash
sudo nano /etc/nginx/sites-available/cre-automator
```
Paste this block inside:
```nginx
server {
    listen 80;
    server_name YOUR_DOMAIN_OR_IP;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```
Enable the configuration and restart Nginx:
```bash
sudo ln -s /etc/nginx/sites-available/cre-automator /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx
```

## 6. Critical Cloudflare & Proxy Setup

> [!WARNING]
> DigitalOcean's IP addresses are flagged as standard commercial datacenters. Because of this, Australian real estate websites protected by Cloudflare will immediately block or lock out the scraper if run directly from the Droplet IP.
>
> To ensure reliability in production, you **MUST** configure residential proxies.
>
> 1. Sign up for a residential proxy service (e.g., Bright Data, Smartproxy, or Webshare).
> 2. Purchase an **Australian Residential Proxy Pool**.
> 3. Edit `backend/config.py` to add proxy credentials, and update the Chrome configuration options in `backend/scraper.py` and `backend/enquiry_agent.py` to route all browser automation through the proxy server.
